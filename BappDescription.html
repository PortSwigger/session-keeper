Session Keeper maintains active sessions with minimal impact by automatically resending low-impact requests at configurable intervals. This extension is ideal for keeping sessions alive during extended breaks or when stepping away from testing, as long as the server supports persistent sessions.

Features

  • Automatically resends requests at custom intervals to maintain session validity
  • Manages multiple sessions simultaneously with independent tab-based session keepers
  • Configurable request intervals and optional maximum request limits
  • Real-time status tracking with countdown timers and response history
  • Editable tab names with visual status indicators ([RUN] when active, [STOP] when paused)

Usage

  1. Identify a low-impact request that maintains your session (such as server time, ping/pong, or a simple API request)
  2. Right-click the request and select "Extensions → Session Keeper → Send to Session Keeper"
  3. Navigate to the Session Keeper tab
  4. Set your desired interval in seconds and optionally specify a maximum number of requests
  5. Click Start to begin automatically sending the request at the configured interval
