Session Keeper is a simple and quick way to keep your session alive in Burp Suite with minimal impact on your requests. This tool is perfect for those instances when you need to keep a session active while away from your desk or even when taking a break or a nap. You can ensure the session stays alive for as long as necessary, but remember that an everlasting session must be supported on the server-side.

Features

  • Automatically resends a low-impact request to maintain active sessions
  • Runs multiple session keepers simultaneously using a tab-based system, with each tab managing an independent request and interval and displaying a status indicator: RUN when active, STOP when paused.
    • Each tab maintains its own request and interval independently
    • Tab names can be edited for easy identification
    • Status indicators show whether the session keeper is active or paused
  • Supports custom intervals and maximum request limits
  • Shows the current request, the response, and the history of requests
  • Provides a Clear Log button to reset the history view
  • Features an easy-to-use UI within Burp Suite

Usage

Start by identifying a low-impact request that has a session you want to keep alive, such as a server time request, a simple API request, or a ping/pong request. Right-click the request and navigate to Extensions → Session Keeper → Send to Session Keeper.

Next, go to the Session Keeper tab and determine the frequency for sending the low-impact request. You can set a maximum number of requests if you do not want to send them indefinitely and then click start.

Session Keeper will start sending the request at the interval you set. Its important to note that the processing time of the request is not included in the countdown timer.

The top window displays the current request being sent, the middle window shows the response, and the bottom window presents a history of the requests. If you need to reset the request history window, use the Clear Log button at the top.

With Session Keeper, you can keep multiple sessions active simultaneously, thanks to its tab-based system.
